## Universal effect rules

- Every flavour event must update at least one visible mechanic or state task.
- Every recurring flavour family must have cooldowns, active caps, and cleanup flags.
- Route variants must change more than text. They need different costs, values, state targets, AI behavior, or follow up missions.
- Repeated failures must escalate into a mission, idea stage, BOP crisis, or foreign reaction instead of repeating the same popup.
- Successful handling should leave memory through lower future cost, safer mission conditions, integration progress, or a better idea stage.
